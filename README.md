# えんぴつだいあろーぐ。— コンセプト
---
ほんの数行のテキストを読むため、
たった1枚の写真を見たいがため、
無数の不要な広告、過剰なアニメーション演出、
大きすぎるサイズの画像など、何かとノイズの多い昨今のWebサイトに対し、
限りなくシンプルで、容量も少なく、10年後20年後でも「インターネットの片隅に残ってしまうような場所」をコンセプトに作成した、実験的なサイトです。

つぶやきでも、対話でもない、でも同じ空間にゆるく共存してるような
ゆるくつながりが感じられるような空間を目指しています。

**公開URL:** `https://kentam1127g.github.io/001/`

### 読まれた感触

ある程度画面に表示された上で、数秒留まって初めて「既読」が増えます。いいねの数も返信機能もありません。インターネット黎明期に感じていた、「この世界の誰かがそこにいた」というゾワゾワするような感触・気配を感じられるようにしました。

### すれ違い

30分以内に誰かが読んでいたら、「読者とすれ違いました」というモーダルが現れます。そこには、その人の名前と一言だけが残されています。また、同様に次の人へ一言のメッセージを残す事ができます。ちょっとだけドキッとするような、偶然の遭遇を表現しました。

### ふんわりとした時間表示とあしあと

投稿時間は1時間単位、誰かのあしあとも「ちょっと前」というアバウトな表示。「あ、誰かが読んでいたんだな」という気配だけ感じられます。

### 通信料を記載

右上、「XXX KB」と表示されている箇所に、通信量がわかるウィンドウを置いています。
クリックすると、スマホの写真何枚分か、動画が何分見られるか、初代マリオブラザーズ何本分に匹敵する通信量かわかります。パケ死対策もバッチリです。

### 器としての設計

データベースではなく、あえてファイルを残すシンプルな設計にしました。もしサービスが終了したとしても、この世にパソコンが存在し、JSONファイルが閲覧できる限りは中身にアクセスできます。

## 技術スタック

| 技術 | 選んだ理由 |
|---|---|
| **Vanilla JS（ビルドレス）** | フレームワーク不要。依存関係ゼロで10年後も動く |
| **GitHub Pages** | 静的配信、CDN・SSL込み、費用ゼロ。日記帳にサーバー代は要らない |
| **Decap CMS ＋ GitHub** | 投稿が JSON ファイルとして git にコミットされる。DBなし、サービス終了後もデータが手元に残る |
| **Netlify Functions ＋ Blobs** | 「誰かが来た気配」だけのための最小 API。フルDBは要らない |
| **GitHub Actions ＋ sharp** | 書き手がファイルサイズを気にしなくていいよう、投稿時に自動で圧縮 |

## 投稿データ構造

個別投稿ファイル（`content/posts/YYYY-MM-DD-HH-MM-SS.json`）のフィールド：

```json
{
  "id": "2026-04-02-14-04-43",
  "author": "まつけん",
  "text": "本文",
  "image": "/images/uploads/xxx.jpg",
  "date": "2026-04-02",
  "createdAt": "2026-04-02T14:04:43+09:00",
  "viewCount": 0
}
```

> `viewCount` はCMSフィールドとして存在するが、**実際の閲覧数はバックエンド（Netlify Blobs）が正値**。このフィールドは参照しない。

## JavaScriptの動作概要

### 初期化フロー（`main.js`）

1. URLハッシュ（`#entry-{id}`）を解析してディープリンク対応
2. `content/posts/index.json` を取得して全投稿をロード
3. バックエンドAPIから閲覧数・最終閲覧日時を取得（`data.js`）
4. 前回訪問時との比較で「新着投稿」を検知 → モーダル表示
5. 他の閲覧者が5分以内に閲覧していた場合「読者とすれ違いました」モーダル表示

### 閲覧数カウント（`render.js`）

IntersectionObserver により以下の条件でカウント：

- 投稿が **60%以上** 表示されている
- その状態が **1.8秒以上** 継続する

条件を満たすとバックエンドAPIの `/counts/bump` を呼び出し、LocalStorage の閲覧済みリストに追加して二重カウントを防止する。

### ローカルストレージキー

| キー | 用途 |
|---|---|
| `quiet-broadcast-seen-entries-v1` | 閲覧済みエントリIDの配列（二重カウント防止） |
| `enpitu-last-latest-id` | 前回訪問時の最新投稿ID（新着検知用） |
| `enpitu-last-read-id` | 最後に閲覧したエントリID |

### 最終閲覧日時の表示フォーマット

| 経過時間 | 表示テキスト |
|---|---|
| 5分未満 | 最終表示：たった今 |
| 31分未満 | 最終表示：ちょっと前 |
| 60分未満 | 最終表示：30分くらい前 |
| 24時間未満 | 最終表示：Xh時間前 |
| 24時間以上 | 最終表示：X日前 |

## Decap CMS 管理画面（`admin/`）

### 認証

GitHub OAuth を使用。認証サーバーは別リポジトリ（`oath/`）が担当。

- **認証ベースURL:** `https://enpitumark-oath.netlify.app`

### 自動フィールド付与（`preSave` フック）

記事保存時に以下を自動設定：

| フィールド | 設定値 |
|---|---|
| `author` | GitHubログイン名から自動取得（`AUTHOR_MAP` で日本語名に変換） |
| `createdAt` | 保存時の現在時刻（JST: `Asia/Tokyo`） |
| `date` | `createdAt` から自動導出 |

### AUTHOR_MAP（ログイン名 → 表示名）

| GitHubログイン | 表示名 |
|---|---|
| `kentam1127g` | まつけん |
| `wakako38-dev` | わかこ |

## デプロイ

GitHub Actions により自動デプロイ：

1. Decap CMS で記事を保存 → GitHub にコミット
2. `scripts/generate-post-index.mjs` が `content/posts/index.json` を再生成
3. GitHub Pages へデプロイ
